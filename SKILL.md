# SCSE 论文/报告格式套用

将此规范应用于武汉大学国家网络安全学院（SCSE）的课程论文和实验报告的内容排版。

## 核心原则

- 仅管理**内容排版格式**，不生成封面页
- 支持两种文档类型：**课程论文** (`paper`) 和 **实验报告** (`report`)
- 用户提供内容（文本、Markdown 或口述），skill 负责将其格式化为符合规范的 .docx
- **论文摘要默认仅生成中文**；仅当用户明确要求英文摘要时才添加 `abstract_en` / `keywords_en`
- **报告不含摘要、关键词、参考文献**，仅包含六大实验模块
- 若用户已有 .docx 初稿，skill 负责检查和修正格式使其符合规范

---

## 使用方式

### 课程论文

```bash
python scripts/format_paper.py -t paper -o <输出.docx> -c <内容.json>
```

### 实验报告

```bash
python scripts/format_paper.py -t report -o <输出.docx> -c <内容.json>
```

`-t` 默认为 `paper`，不指定时按课程论文处理。

## 内容 JSON 结构

### 论文结构 (`-t paper`)

```json
{
    "abstract_cn": "中文摘要正文…",
    "keywords_cn": ["关键词1", "关键词2", "关键词3"],
    "sections": [
        {
            "title": "1 绪论",
            "level": 1,
            "content": ["段落一。", "段落二。"],
            "subsections": [
                {"title": "1.1 研究背景", "level": 2, "content": ["…"]}
            ]
        }
    ],
    "references": [
        "[1] 作者. 文章题目[J]. 期刊名, 出版年份, 卷号(期数):起止页码."
    ]
}
```

- **摘要默认仅生成中文**，只有用户明确要求时才加入英文摘要（`abstract_en` 和 `keywords_en` 字段）
- `sections` 中 `title` 需包含编号（如「1 绪论」），脚本不自动编号
- 参考文献为可选

### 报告结构 (`-t report`)

报告不含摘要、关键词和参考文献，仅由六大模块组成：

```json
{
    "title": "实验报告标题",
    "purpose": {
        "title": "实验目的",
        "content": ["本次实验旨在…"]
    },
    "environment": {
        "title": "实验环境",
        "content": ["操作系统：Windows 11", "开发工具：Python 3.12, VS Code", "依赖库：numpy, pandas, matplotlib"]
    },
    "experiment_content": {
        "title": "实验内容",
        "content": ["本实验主要包括以下内容：", "1. 数据集预处理", "2. 模型构建与训练", "3. 结果评估与分析"]
    },
    "steps": {
        "title": "实验步骤",
        "content": ["步骤一：加载并预处理数据…", "步骤二：构建神经网络模型…"]
    },
    "results": {
        "title": "实验结果与分析",
        "content": ["实验结果表明…", "从结果可以看出…"]
    },
    "summary": {
        "title": "实验总结",
        "content": ["通过本次实验…"]
    }
}
```

- `title` 为报告总标题（黑体二号居中），可选
- 六个模块按上述固定顺序渲染，每个模块为一级标题（黑体小二号）
- 每个模块的 `title` 字段可自定义标题文本，`content` 为字符串数组，每项为一个段落
- 报告**不含摘要、关键词、参考文献**

## 排版规范

以下规范源自《武汉大学国家网络安全学院课程论文书写及排版规范》，论文和报告均适用。

### 页面设置

| 项目 | 规格 |
|------|------|
| 纸张 | A4（210mm × 297mm） |
| 上边距 | 25mm |
| 下边距 | 20mm |
| 左边距 | 30mm |
| 右边距 | 30mm |
| 页码 | 页脚居中，Times New Roman 5号（10.5pt），阿拉伯数字连续编码 |
| 装订 | 左侧装订 |

### 字体与字号

| 元素 | 字体 | 字号 | 对应磅值 |
|------|------|------|----------|
| 论文/报告题目 | 黑体 | 2号 | 22pt |
| 各章标题 | 黑体 | 小2号 | 18pt |
| 各节一级标题 | 黑体 | 4号 | 14pt |
| 各节二级标题 | 黑体 | 小4号 | 12pt |
| 各节三级标题 | 黑体 | 小4号 | 12pt |
| 款项 | 黑体 | 小4号 | 12pt |
| **正文** | **宋体** | **小4号** | **12pt** |
| 摘要/结论/参考文献 标题 | 黑体 | 小2号 | 18pt |
| 摘要/结论/参考文献 内容 | 宋体 | 小4号 | 12pt |
| 英文摘要标题 | Times New Roman 大写粗体 | 小2号 | 18pt |
| 英文摘要内容 | Times New Roman | 小4号 | 12pt |
| 中文关键词标题 | 黑体 | 小4号 | 12pt |
| 中文关键词 | 宋体 | 小4号 | 12pt |
| 英文关键词标题 | Times New Roman 粗体 | 小4号 | 12pt |
| 英文关键词 | Times New Roman | 小4号 | 12pt |
| 目录标题 | 黑体 | 小2号 | 18pt |
| 目录中章标题 | 黑体 | 4号 | 14pt |
| 目录中其他内容 | 宋体 | 小4号 | 12pt |

### 段落间距

| 元素 | 段前 | 段后 | 行距 |
|------|------|------|------|
| 章标题 | 0.8行 | 0.5行 | — |
| 节标题 | 0.5行 | 0.5行 | — |
| 正文 | — | — | 固定值 23磅 |
| 正文首行缩进 | — | — | 2字符 |

### 标题编号规则

- 使用「1……、1.1……、1.1.1……」格式，**不使用**「第一章」等中文序数
- 条以下款项依次用「1.1.1.1」「(1)」「①」标识
- 各级标题不使用标点符号
- 目录和正文标题题序须一致，题序中阿拉伯数字用 Times New Roman 字体
- **每章结束后另起一页**

### 摘要与关键词

- 中文摘要 300-500 字，关键词 3-5 个，用「；」分隔，最后一个关键词不加标点
- 英文摘要与中文摘要内容对应，约 300 实词
- 摘要正文下空一行顶格打印「关键词」款项
- 英文摘要另起一页

### 正文内容规范

以下为格式生成和排版检查时的参考规范：

1. **引文标示** — 方括号上标，如「……模式[3]」，用小4号字体。多处引用并列如「文献[1,2,6-9]」。各级标题不得使用引文标示。
2. **物理量符号** — 变量符号用斜体，计量单位符号用正体。
3. **数字** — 一般用阿拉伯数字。年份用4位数字。小于1的小数加前导0（相关系数、p值等例外：r=.26, p<.05）。
4. **公式** — 居中书写，序号按章编排如「(3.2)」。公式较长时在「＝」或运算符号处转行。
5. **表格** — 表标题和表序号居中置于表上方（黑体小4号）。序号按章编排如「表2.4」。三线表格式。
6. **图** — 图标题和图序号居中置于图下方（宋体小4号）。序号按章编排如「图1.4」。
7. **注释** — 页末注或篇末注，宋体5号，用「①、②」标注。

### 参考文献格式（GB/T 7714-2005）

序号左顶格加方括号，如 `[1]`。每条以 `.` 结束。常见类型：

- **期刊** [J]：`[序号] 作者. 文章题目[J]. 期刊名, 出版年份, 卷号(期数):起止页码.`
- **图书** [M]：`[序号] 作者. 书名[M]. 版次. 出版地: 出版单位, 出版年份: 起止页码.`
- **会议论文** [C]：`[序号] 作者. 文章题目[A]. 主编. 论文集名[C], 出版地: 出版单位, 出版年份: 起止页码.`
- **学位论文** [D]：`[序号] 作者. 论文题目[D]. 保存地: 保存单位, 年份.`
- **报告** [R]：`[序号] 报告者. 报告题目[R]. 报告地: 报告会主办单位, 报告年份.`
- **专利** [P]：`[序号] 专利所有者. 专利名称: 专利国别, 专利号[P]. 发布日期.`
- **标准** [S]：`[序号] 标准代号. 标准名称[S]. 出版地: 出版单位, 出版年份.`
- **报纸** [N]：`[序号] 作者. 文章题目[N]. 报纸名, 出版日期(版次).`
- **电子文献**：`[序号] 作者. 文献题目[文献类型/载体类型]. 可获取地址, 发表或更新日期/引用日期.`

## 格式修正模式

当用户要求检查或修正已有文档格式时：

1. 读取文档内容，逐项对照上述规范
2. 修正页边距、字体、字号、行距、段落间距
3. 修正标题层级及编号格式
4. 修正参考文献引用格式
5. 不修改内容措辞，仅调整排版属性
6. 输出修正后的 .docx
